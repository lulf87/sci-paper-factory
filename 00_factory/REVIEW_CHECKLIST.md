# REVIEW_CHECKLIST.md

## 0. 用途

本清单用于投稿前终审。  
目标不是“把稿子修漂亮”，而是确认：

- 没有编造
- 没有越界
- 没有不可复现结果
- 没有期刊不匹配
- 没有把公开数据 + 本地仿真的边界写穿

任一关键项不通过，均不得投稿。

---

## 1. 一票否决项（必须全部通过）

以下任一项为“否”，稿件状态直接为 `DO NOT SUBMIT`。

| 编号 | 关键问题 | 通过(是/否) | 备注 |
|---|---|---|---|
| K1 | 所有核心结果都能追溯到真实数据或真实脚本输出 |  |  |
| K2 | 不存在虚构文献、虚构数据、虚构验证、虚构统计结果 |  |  |
| K3 | 摘要、结果、结论中不存在越界主张 |  |  |
| K4 | 所有图表均可由脚本重现，不依赖手工改数 |  |  |
| K5 | 仿真边界、假设、参数来源均已写清 |  |  |
| K6 | 期刊 scope、作者指南、稿型已核验 |  |  |
| K7 | 所有占位符、TODO、伪引用均已清除 |  |  |
| K8 | 若无外部验证，则稿件未暗示“已真实验证/可部署/临床有效” |  |  |

## 2. 研究身份与范围

### 2.1 研究问题
- [ ] 研究问题能用一句话准确表述
- [ ] 研究对象、输入、输出、主要终点定义一致
- [ ] 研究边界与非研究边界已明确
- [ ] 论文标题未夸大研究范围
- [ ] 摘要首段未虚构研究动机或“重大空白”

### 2.2 贡献陈述
- [ ] 贡献点与实际完成内容一致
- [ ] 未将“计划做/可扩展”写成“已完成贡献”
- [ ] 未将工程整理工作包装成方法创新
- [ ] 未将仿真解释包装成机制证实

---

## 3. 数据来源与合规

### 3.1 数据真实性
- [ ] 每个数据源都有明确名称
- [ ] 每个数据源都有获取路径或记录
- [ ] 每个数据源都有版本号/发布日期/下载日期（若可得）
- [ ] 数据字典或字段说明已保存
- [ ] 样本纳入排除标准已固定
- [ ] 数据清洗规则已文档化

### 3.2 合规与许可
- [ ] 数据许可已检查
- [ ] 使用方式未超出许可
- [ ] 若有人体/敏感信息，伦理与隐私处理已说明
- [ ] 未使用来源不明数据
- [ ] 未把闭源数据伪装成公开数据

### 3.3 数据质量
- [ ] 缺失值处理有明确定义
- [ ] 异常值处理有明确定义
- [ ] 重复样本处理有明确定义
- [ ] 训练/验证/测试划分无信息泄漏
- [ ] 结果中样本量与方法中一致

---

## 4. 仿真模块审查

### 4.1 仿真定义
- [ ] 仿真目标明确
- [ ] 状态变量、输入、输出已定义
- [ ] 仿真与公开数据变量映射关系已列出
- [ ] 参数来源逐项记录
- [ ] 不可观测参数已明确标记为假设或待校准项

### 4.2 仿真合理性
- [ ] 仿真假设表已写入正文或补充材料
- [ ] 校准方法已说明
- [ ] 未校准项已说明
- [ ] 灵敏度分析已执行，或明确承认未执行
- [ ] 不存在“仿真结果 = 现实已验证结果”的写法

### 4.3 仿真复现
- [ ] 仿真代码可运行
- [ ] 随机种子固定
- [ ] 配置文件保留
- [ ] 输出目录保留
- [ ] 关键图表可重跑

---

## 5. 方法与统计

### 5.1 方法描述
- [ ] 方法足以让他人复现
- [ ] 关键超参数已披露
- [ ] 基线方法选择有理由
- [ ] 对照设置公平
- [ ] 未遗漏失败条件与边界条件

### 5.2 统计与评估
- [ ] 每个指标定义清楚
- [ ] 指标选择与研究问题一致
- [ ] 统计检验方法已说明（若使用）
- [ ] 未把“数值更高”写成“显著更优”除非有统计支持
- [ ] 置信区间、标准差、重复次数等写法一致
- [ ] 未把一次性最佳结果当作稳定性能
- [ ] 稳健性/敏感性检查已做或如实说明未做

### 5.3 消融与对照
- [ ] 若声称某模块有效，已提供适当对照
- [ ] 若未做消融，正文未过度强调模块贡献
- [ ] 若无外部验证，正文未用“validated”笼统表述

---

## 6. 结果完整性

### 6.1 结果可追溯
- [ ] 每个表格都有对应脚本/产物路径
- [ ] 每张图都有对应脚本/产物路径
- [ ] 论文中的每个关键数值都可定位到源文件
- [ ] 无“手改后未同步”的版本冲突
- [ ] 不同章节中的同一数值前后一致

### 6.2 结果表达
- [ ] 结果节只报告观察到的结果
- [ ] 解释性内容放在讨论节
- [ ] 未隐藏与主叙事不一致的重要结果
- [ ] 失败实验若影响结论，已如实说明
- [ ] 未使用选择性展示造成误导

---

## 7. 主张边界审查

### 7.1 逐句分类
- [ ] 核心句已在内部完成 E/I/H/X 分类
- [ ] 摘要中无 X
- [ ] 结果中无 I/H/X
- [ ] 结论中无 X
- [ ] 所有核心主张均有 claim ledger 记录

### 7.2 常见越界排查
- [ ] 未将关联写成因果
- [ ] 未将仿真写成现实证明
- [ ] 未将内部实验写成外部验证
- [ ] 未将局部数据写成普遍规律
- [ ] 未将假设写成事实
- [ ] 未将可行性写成可部署性
- [ ] 未将 retrospective/public 数据结果写成 clinical utility

### 7.3 用语排查
- [ ] 已删除或替换高风险词：prove / confirm / causal / deployment-ready / clinically effective
- [ ] 保守措辞使用恰当：suggest / may / consistent with / under current assumptions
- [ ] 结论范围不超过数据范围和仿真假设范围

---

## 8. 文献与引用

### 8.1 引文真实性
- [ ] 每条参考文献都真实存在
- [ ] 关键文献已实际核验
- [ ] 引文元数据完整
- [ ] 正文中的引用与参考文献一一对应
- [ ] 无孤儿引用、无未引用条目

### 8.2 引用准确性
- [ ] 未凭印象转述文献
- [ ] 未夸大他人结论
- [ ] 未使用二手引述冒充原始证据
- [ ] “已有研究表明”后面均有真实支持
- [ ] 综述性表述未超出实际文献覆盖范围
