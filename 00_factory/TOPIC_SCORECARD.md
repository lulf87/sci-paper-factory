# TOPIC_SCORECARD.md

## 0. 用途

本表用于在立项前快速筛选题目，适用于：

- 公开数据
- 本地仿真
- 以 SCI/SCIE 原创研究为目标

核心原则：

- 不做无法验证的题
- 不做必须依赖私有数据的题
- 不做主结论必须依赖额外湿实验/临床试验/现场部署才能成立的题
- 不做只能靠“话术放大”才能发表的题

## 1. 一票否决项（任一项为“是”即直接拒绝）

请逐项回答：

| 编号 | 问题 | 是/否 |
|---|---|---|
| G1 | 主问题必须依赖私有、封闭、不可复核数据？ |  |
| G2 | 关键标签/终点无法从公开数据中可靠获得？ |  |
| G3 | 主结论必须依赖湿实验、前瞻性试验、真实部署验证才成立？ |  |
| G4 | 仿真状态变量与公开数据变量无法建立清晰映射？ |  |
| G5 | 研究设计天然会诱导因果越界，但又无法加以限制？ |  |
| G6 | 数据许可、伦理、合规存在明显障碍？ |  |
| G7 | 只能复刻公开 benchmark，几乎没有新增问题/方法/分析层价值？ |  |
| G8 | 预期论文必须编造或过度包装验证链才能成立？ |  |

**判定规则：**

- 任一项为“是” → `REJECT`
- 全部为“否” → 进入加权评分

## 2. 加权评分表

评分范围统一为 `0–5`：

- `0` = 不成立/严重缺陷
- `1` = 勉强可做，但高风险
- `2` = 可做，存在明显短板
- `3` = 基本可行
- `4` = 较强
- `5` = 很强，具备投稿潜力

加权得分公式：

`Weighted Score = Σ(单项得分 / 5 × 权重)`

总分满分 `100`。

| 编号 | 维度 | 权重 | 评分说明（看 3 节） | 得分(0-5) | 加权分 | 证据/备注 |
|---|---:|---:|---|---:|---:|---|
| S1 | 科学问题清晰度 | 10 | 问题是否可被一句话定义并可被数据/仿真回答 |  |  |  |
| S2 | 公开数据可得性与完整性 | 12 | 数据是否真实可得、字段齐全、版本明确 |  |  |  |
| S3 | 数据—仿真映射质量 | 10 | 仿真变量是否能被公开数据约束/校准 |  |  |  |
| S4 | 终点/标签可观测性 | 8 | 关键结果是否能被可重复测量 |  |  |  |
| S5 | 基线与对照可实现性 | 6 | 是否有合理 baseline/对照方案 |  |  |  |
| S6 | 新颖性与增量价值 | 10 | 相对现有公开工作是否有明确新增价值 |  |  |  |
| S7 | 方法复杂度与计算可承受性 | 8 | 本地资源能否完成主要实验与仿真 |  |  |  |
| S8 | 验证路径充分性 | 10 | 是否有真实、可执行、非编造的验证路径 |  |  |  |
| S9 | 越界风险可控性 | 8 | 是否能在不夸大的前提下形成成立的结论 |  |  |  |
| S10 | 期刊匹配广度 | 6 | 是否存在多个潜在期刊方向 |  |  |  |
| S11 | 写作与复现工作量可控性 | 5 | 是否能形成完整稿件与复现包 |  |  |  |

## 3. 各维度评分锚点

### S1 科学问题清晰度（10）

- `0`：问题含糊，无法定义主要变量
- `1`：问题可表述，但无法转为可检验目标
- `3`：问题基本明确，可定义主要结果
- `5`：问题明确、边界清楚、可直接映射到研究设计

### S2 公开数据可得性与完整性（12）

- `0`：无可靠公开数据
- `1`：有数据但核心字段缺失或不可下载
- `3`：有可用数据，但清洗成本较高或字段不完全
- `5`：数据来源稳定、版本明确、核心字段齐全、许可清楚

### S3 数据—仿真映射质量（10）

- `0`：无映射关系
- `1`：只能类比，无法约束参数
- `3`：部分变量可映射，可做有限校准
- `5`：核心状态变量、输入、输出均有明确映射路径

### S4 终点/标签可观测性（8）

- `0`：核心终点不可观测
- `1`：终点依赖高主观性或不可复核标注
- `3`：终点可得但定义有噪声
- `5`：终点定义清晰、可重复、误差可解释

### S5 基线与对照可实现性（6）

- `0`：无合理对照
- `1`：对照不可复现或成本过高
- `3`：有可实现 baseline，但质量一般
- `5`：有清晰、可复现、社区认可的 baseline/对照组

### S6 新颖性与增量价值（10）

- `0`：纯重复，无新增价值
- `1`：仅参数微调或换数据重跑
- `3`：有局部增量，如新耦合框架/新分析视角
- `5`：问题、方法或验证结构均有明确新增贡献

### S7 方法复杂度与计算可承受性（8）

- `0`：现有资源无法完成
- `1`：理论可做，但资源严重不足
- `3`：主要实验可完成，但需严格控规模
- `5`：资源、时间、算力均与方案匹配

### S8 验证路径充分性（10）

- `0`：无可执行验证路径
- `1`：只能依赖口头解释或单一结果
- `3`：有内部验证与基础稳健性检查
- `5`：有完整验证链，如多切分/稳健性/敏感性/外部公开数据对照（若可得）

### S9 越界风险可控性（8）

- `0`：主结论天然会越界
- `1`：必须强行放大结论才能看起来“像论文”
- `3`：通过收窄表述可控制风险
- `5`：即便严格保守表述，论文仍能成立

### S10 期刊匹配广度（6）

- `0`：几乎无适配期刊
- `1`：只适合极窄且高风险投稿口
- `3`：存在少数可尝试期刊
- `5`：有多个明确候选方向与备选层级

### S11 写作与复现工作量可控性（5）

- `0`：工作量失控
- `1`：依赖过多手工、不可复核流程
- `3`：可完成，但需要严格项目管理
- `5`：可稳定产出稿件、附录、代码与复现包

### S12 伦理/许可/IP 安全性（7）

- `0`：明显不合规
- `1`：许可或伦理不清晰
- `3`：有少量待核验条款
- `5`：许可明确、合规风险低、披露路径清楚

## 4. 分数解释与动作建议

| 总分 | 判定 | 动作 |
|---:|---|---|
| 85–100 | `GO_STRONG` | 进入正式方案冻结与期刊映射 |
| 75–84 | `GO_CONDITIONAL` | 可做，但需先补齐短板并锁定边界 |
| 65–74 | `HOLD` | 仅在存在明确修正路径时继续 |
| <65 | `REJECT` | 不建议投入写作资源 |

### 强制附加规则

即使总分较高，只要满足以下任一项，也不得进入写稿：

1. `S8 验证路径充分性 < 3`
2. `S9 越界风险可控性 < 3`
3. `S2 公开数据可得性与完整性 < 3`
4. 任一一票否决项触发

## 5. 推荐的选题判断顺序

按以下问题顺序快速判断：

1. **问题是否成立？**  
   能否用一句话说明“研究什么、为什么重要、结果是什么”

2. **数据是否成立？**  
   是否真的存在公开数据支撑主要结果

3. **仿真是否成立？**  
   是否只是“画一个模型”，还是能被数据约束

4. **验证是否成立？**  
   是否能不编造地形成验证链

5. **结论是否成立？**  
   即便严格保守表达，是否仍有可投稿价值

只有 1–5 全部成立，才值得继续。
